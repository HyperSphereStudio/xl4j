Introduction

This add-in is intended to demonstrate the type of add-in you can build using XL4J. It demonstates a broad range of functionality split broadly into purely financial and general purpose.

Financial Features

These features are used to demonstrate how specific financial applications can be built. It is not intended to be exhaustive, but simply to support the demonstration sheets we have created.

  • Pulling data from the Quandl data service
  • Processing Quandl data into Time Series objects
  • Schedule generation for time series sampling
  • Time series manipulation
    • arithmetic operations
    • padding
    • statistics (variance, covariance, correlation, correlation matrices)
    • sampling using generated schedules
    • a variety of returns calculators
  • Credit Default Swaps
    • CDS Trade object creation
    • Compute Accrual start and end dates/times; Annuity payments, Payment dates/times
    • ISDA standard credit and yield curve creation
    • ISDA standard analytics, pricing per leg, hazard rates, survival probabilities, interpolation on curves, etc.
    • More flexible than the ISDA CDS Excel add-in
  • Supporting functions
    • Matrix expansion
    • Schedule expansion

General purpose features

Thes features are applicable those both inside and outside the financial services sector.

  • Asynchronous REST queries
  • JSON Decoding and object drilldown, query and expansion
  • Dynamic Java invocation
    • Create objects by calling constructors
    • Call static and dynamic methods
    • Create enum instances
    • Extract static and dynamic field values
    • Chain method calls
    • Create arrays (primitive and not) and containers
    • Choose simplest or object reference return types (the X variant of functions always return object references)
    • Automatic type marshalling
    • Use can be mixed with custom functions

The Sample Sheets

You can find the sample sheets in the examples sub-directory

Filename Description
rest-countries.xlsx Calls out to a REST service provided by groupkt.com that returns country code information via JSON. It demonstrates constructing a request, executing it, and drilling down into the JSON response to extract the useful data.
swing-image.xlsxThis is a slightly silly demonstration that writes the pattern set into a grid of Excel cells and creates a Swing toolkit window reflecting the pattern. It is just meant to show the use of the Java primitives and does not deal with issues like multi-threading access to Swing components in a serious way.
sovereign-cds-pricing-and-risk.xlsx A more serious example showing a portfolio of Soverign (although it could be readily adapter to non-soverign CDS with the appropriate data) CDS pricing and risk. Both yield and credit curves are created from real (historical, hardcoded) market data, CDS are priced and have both credit and interest rate risk calculated in bucketed form and displayed both per-trade and per-currency. Various other analytics are also displayed.
time-series-and-efficient-frontier.xlsm A long-only equity portfolio analysis using real data pulled from the Quandl data service (refresh requires a free sign-up and API key, instructions on first sheet of workbook). Pulls historical data for a sample portfolio, computes mean return, variance, standard deviation and then calculates covariance, correlation and CAPM Beta against the index. We then use VBA to drive the standard Excel Solver add-in (which will need to be enabled if you want to recalculate) to optimise a selection of portfolios for minimum risk, given a set target return for each one. Using this, we create an 'efficient frontier' graph showing the risk/return trade-off. There's also a test portfolio allowing you to show a specific portfolio within this. Lastly as a sanity check, we optimize a portfolio to maximise it's Sharpe ratio. Drawing a line through this point and the origin should be at a tangent to the edge of the frontier. To run the solver, you need to enable macros so the sheet can configure the solver automatically when you click the buttons.
